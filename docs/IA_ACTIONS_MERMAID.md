# IA 4번 동작 Mermaid

> 기준 문서: [IA_TEMPLATE_ACTIONS.md](IA_TEMPLATE_ACTIONS.md)의 `4. 동작`  
> 목적: 버튼 클릭, 조건 분기, 상태 변경, 화면 변경, 토스트/확인창 흐름을 Mermaid로 빠르게 확인한다.  
> 작성 기준: 독립 동작은 `4.n.n` 단위로 분리하고, 연속 사용 흐름은 한 다이어그램으로 묶는다.

## 4.1 미리보기 동작

```mermaid
flowchart TD
  A["미리보기 썸네일 클릭"] --> B["클릭한 slideId를 선택 슬라이드로 저장"]
  B --> C["클릭한 moduleId를 선택 모듈로 저장"]
  C --> D["현재 PPT 정보 변경"]
  D --> E["슬라이드 카운터 변경"]
  E --> F["현재 PPT 미리보기 변경"]
  F --> G["제작탭 선택 모듈 하이라이트 변경"]

  H{"슬라이드가 있는가?"}
  H -- "없음" --> I["미리보기: 모듈을 추가하세요"]
  I --> J["카운터: 0 / 0"]
  H -- "있음" --> A

  K["모듈 순서 변경"] --> L["썸네일 그룹 순서 변경"]
```

## 4.2 제작탭 - 모듈 동작

### 4.2.1 모듈 선택 시

```mermaid
flowchart TD
  A["모듈 선택"] --> B["선택 모듈 ID 저장"]
  B --> C["해당 모듈 첫 번째 슬라이드 선택"]
  C --> D["선택 모듈 카드 하이라이트"]
  D --> E["미리보기 영역을 첫 번째 슬라이드로 변경"]
```

### 4.2.2 모듈 추가 버튼 클릭 시

```mermaid
flowchart TD
  A["모듈 추가 버튼 클릭"] --> B["클릭한 타입의 새 모듈 생성"]
  B --> C["모듈 목록 마지막에 추가"]
  C --> D["새 모듈을 현재 선택 모듈로 변경"]
  D --> E["새 모듈 첫 번째 슬라이드 선택"]
  E --> F["전체 모듈 수/PPT 장 수 갱신"]
  F --> G["미리보기 썸네일 목록 갱신"]
  G --> H["토스트: {모듈 종류} 모듈을 추가했습니다."]
```

## 4.6 제작탭 - 초기화 동작

```mermaid
flowchart TD
  A["초기화 버튼 클릭"] --> B{"현재 저장 템플릿이 있고 변경사항이 있는가?"}
  B -- "예" --> C{"초기화 전에 현재 템플릿에 덮어쓸까요?"}
  C -- "덮어쓰기" --> D["현재 저장 템플릿 갱신"]
  D --> H{"현재 작업을 초기 샘플로 되돌릴까요?"}
  C -- "덮어쓰지 않음" --> E{"변경된 내용을 새 템플릿으로 저장할까요?"}
  E -- "저장" --> F["템플릿 저장 모달 열기"]
  F --> G["저장 완료"]
  G --> H
  E -- "저장 안 함" --> H
  B -- "아니오" --> H

  H -- "취소" --> I["초기화하지 않음"]
  H -- "확인" --> J["기본 상태로 초기화"]
  J --> K["제작탭으로 이동"]
  K --> L["템플릿 이름: 새로운 템플릿"]
  L --> M["기본 모듈: 찬양 1개, 성경 1개"]
  M --> N["첫 번째 모듈 선택"]
  N --> O["PPT 설정 기본값 복원"]
  O --> P["토스트: 초기 샘플로 되돌렸습니다."]
```

## 4.7 템플릿 저장 동작

```mermaid
flowchart TD
  A["템플릿 저장 버튼 클릭"] --> B{"로그인 상태인가?"}
  B -- "아니오" --> C["로그인 화면 이동"]
  C --> D["pending action = save 저장"]
  D --> E["로그인 완료"]
  E --> F["저장 흐름 재개"]
  B -- "예" --> F

  F --> G{"기존 저장 템플릿에서 시작한 작업인가?"}
  G -- "예" --> H{"{템플릿 이름} 템플릿에 덮어쓸까요?"}
  H -- "확인" --> I["기존 템플릿 덮어쓰기"]
  I --> J["토스트: 템플릿을 덮어썼습니다."]
  H -- "취소" --> K["새 템플릿 저장 모달 열기"]
  G -- "아니오" --> K

  K --> L["템플릿 이름 기본값 입력"]
  L --> M["입력창 포커스/텍스트 선택"]
  M --> N["저장 버튼 클릭"]
  N --> O["입력 이름으로 새 템플릿 생성"]
  O --> P{"이름이 비었는가?"}
  P -- "예" --> Q["이름 없는 템플릿으로 저장"]
  P -- "아니오" --> R["입력 이름 사용"]
  Q --> S["중복 이름이면 번호 자동 부여"]
  R --> S
  S --> T["저장 목록 맨 앞에 추가"]
  T --> U["현재 템플릿 이름/ID 갱신"]
  U --> V["토스트: 템플릿을 저장했습니다."]

  K --> W["닫기/취소 클릭"]
  W --> X["모달 닫힘"]
  X --> Y["불러오기/초기화 중 저장 흐름이면 대기 작업 취소"]
```

## 작성 메모

- `4.2`, `4.3`, `4.4`, `4.5`, `4.12`, `4.13`은 `4.n.n` 단위로 Mermaid를 분리했다.
- `4.8`은 저장 템플릿 목록/미리보기/삭제 흐름, 추천 템플릿 미리보기, 적용하기 흐름 세 묶음으로 정리했다.
- `4.9`는 설정 진입/PPT 비율을 `4.n.n` 단위로 유지하고, 로고는 `로고 섹션 토글`, `로고 ON 및 이미지 설정`, `로고 OFF` 세 흐름으로 나눴다.
- `4.10`은 로그인/로그아웃 및 pending action 흐름을 하나의 Mermaid로 통합했다.
- `4.1`, `4.6`, `4.7`, `4.11`은 사용자가 지정한 분리 대상이 아니므로 기존처럼 섹션 단위 흐름으로 유지했다.
- 확인창은 조건 노드로 표현했다.
- `pending action`은 로그인 후 이어서 실행되는 작업 흐름으로 표현했다.
