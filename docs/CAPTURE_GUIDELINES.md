# 캡쳐 지시사항

스토리보드용 화면 캡쳐를 만들 때 공통으로 적용할 기준이다.

## 1. 캡쳐 대상 배치

1. 캡쳐할 모듈은 작업영역의 가장 상단에 둔다.
2. 불필요한 다른 모듈 때문에 캡쳐 대상 모듈이 아래로 밀리게 하지 않는다.
3. 캡쳐에 필요 없는 모듈은 만들지 않거나 접힌 상태로 둔다.
4. 토글은 토글 자체를 보여주기 위한 목적이 아니라, 긴 모듈에서 필요한 영역만 보이도록 접고 펼치는 용도로 사용한다.

## 2. 영역 표시

1. 화면에 보이는 영역만 사각형 처리한다.
2. viewport 밖 DOM 영역은 표시하지 않는다.
3. 작업영역 패널의 보이는 경계 밖으로 나간 영역은 잘라낸다.
4. 요소가 일부만 보이면 보이는 부분만 표시한다.
5. 요소가 전혀 보이지 않으면 해당 캡쳐에서는 번호와 사각형을 표시하지 않는다.
6. 한 화면에 안 들어가면 스크롤/토글로 먼저 해결하고, 그래도 안 되면 `_part1`, `_part2`로 분리한다.

## 3. 번호 표시

1. 번호는 해당 IA 항목의 앞 `n.n` 부분을 제외하고 표기한다.
    1. 예: `2.1.3`은 `3`으로 표기한다.
    2. 예: `2.4.3.2`는 `3.2`로 표기한다.
2. 번호는 기본적으로 사각형 우측 상단 모서리 중심에 둔다.
3. 번호끼리 겹치면 주변 빈 공간으로 이동한다.
4. 번호를 이동해도 대상 사각형과 너무 멀어지지 않게 한다.
5. 화면 밖으로 잘리는 번호는 캔버스 여백 안쪽으로 보정한다.

## 4. 시각 스타일

1. 하이라이트 색상과 번호 배지 색상은 기존 파란색 계열을 유지한다.
2. 하이라이트 fill은 UI를 읽을 수 있을 만큼 투명하게 한다.
3. 사각형 내부 텍스트, 버튼, 입력값이 가려지지 않아야 한다.
4. 이미지 위에는 번호만 표시하고, 한글 설명은 문서 표에 작성한다.

## 5. 원본 보관

1. 주석 없는 raw 원본과 주석본을 함께 저장한다.
2. raw 원본은 최종 캡쳐 상태와 동일해야 한다.
3. 주석본을 다시 만들 때도 raw 원본이 같은 화면 상태인지 확인한다.

## 6. 검증

1. 대표 이미지를 직접 열어 확인한다.
2. 대상 모듈이 작업영역 최상단에 있는지 확인한다.
3. 사각형이 작업영역에서 보이는 부분 밖으로 나가지 않는지 확인한다.
4. 번호끼리 겹치지 않는지 확인한다.
5. 한글 문구가 `???`처럼 깨지지 않고 정상 표시되는지 확인한다.
