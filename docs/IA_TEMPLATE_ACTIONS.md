# IA 화면/모듈/템플릿/동작 보강본

> 기준 파일: `docs/IA.md`
> 작성 기준: 현재 `index.html`, `app.js`에 구현된 실제 화면 요소와 이벤트 동작
> 범위: `1. 전체화면구성`, `2. 모듈 설명`, `3. 템플릿 설명`, `4. 동작` 보강

## 1. 전체화면구성 보강

### 1.1 상단바

1. 상단바는 앱 브랜드, 현재 작업 요약, 로그인 상태, 주요 실행 버튼으로 구성된다.
2. 브랜드 영역
    1. `WorshipFlow` 앱 이름을 표시한다.
    2. 현재 작업 요약을 `n개 모듈 · n장` 형식으로 표시한다.
    3. 모듈 수와 슬라이드 수는 모듈 추가, 삭제, 복제, 입력 변경, 템플릿 적용 시 갱신된다.
3. 로그인 상태
    1. 로그아웃 상태이면 `로그인 전`으로 표시한다.
    2. 로그인 상태이면 로그인한 이메일을 표시한다.
4. 로그인/로그아웃 버튼
    1. 로그아웃 상태이면 `로그인` 버튼으로 표시된다.
    2. 로그인 상태이면 `로그아웃` 버튼으로 표시된다.
5. 템플릿 저장 버튼
    1. 현재 작업 상태를 저장 템플릿으로 저장하는 버튼이다.
    2. 제작탭 패널의 `템플릿 저장` 버튼과 같은 저장 흐름을 실행한다.
6. PPT 생성 버튼
    1. 현재 모듈 목록과 설정값 기준으로 PPT 파일을 생성한다.
7. 공유하기 버튼
    1. 현재 템플릿 정보와 공유 링크를 표시하는 공유 모달을 연다.

### 1.2 메뉴

1. 메뉴는 제작, 템플릿, 설정, My 항목으로 구성된다.
2. 제작 메뉴
    1. 예배 순서와 모듈을 편집하는 제작탭으로 이동한다.
3. 템플릿 메뉴
    1. 저장된 템플릿과 추천 템플릿을 확인하는 템플릿탭으로 이동한다.
4. 설정 메뉴
    1. PPT 비율과 로고 설정을 조정하는 설정탭으로 이동한다.
5. My 메뉴
    1. 현재 MVP에서는 비활성 메뉴이다.
    2. 클릭 시 화면 전환 없이 `MVP에서는 제작 메뉴만 사용합니다.` 토스트가 표시된다.

### 1.3 작업영역

1. 작업영역은 선택한 메뉴에 따라 제작탭, 템플릿탭, 설정탭 중 하나를 표시한다.
2. 제작탭
    1. 현재 템플릿 이름 입력창을 표시한다.
    2. 제작탭 내부 `템플릿 저장` 버튼을 표시한다.
    3. 전체 모듈 수를 숫자 배지로 표시한다.
    4. 초기화 버튼을 표시한다.
    5. 모듈 목록을 표시한다.
    6. 모듈 추가 버튼 6종을 표시한다.
        1. 찬양
        2. 찬송가
        3. 성경
        4. 설교
        5. 광고
        6. 커스텀
3. 템플릿탭
    1. 저장된 템플릿 섹션을 표시한다.
    2. 추천 템플릿 섹션을 표시한다.
    3. 템플릿 카드를 클릭하면 목록 대신 템플릿 미리보기 화면을 표시한다.
4. 설정탭
    1. PPT 비율 섹션을 표시한다.
    2. 로고 섹션을 표시한다.
    3. 각 설정 섹션은 토글로 접고 펼 수 있다.

### 1.4 미리보기 영역

1. 미리보기 영역은 현재 선택된 슬라이드를 크게 보여주는 영역이다.
2. 현재 미리보기 중인 PPT 정보
    1. `모듈 이름 · 모듈 내 순번` 형식으로 표시된다.
    2. 슬라이드가 없으면 `슬라이드`로 표시된다.
3. 슬라이드 카운터
    1. `현재 순번 / 전체 PPT 장 수` 형식으로 표시된다.
    2. 슬라이드가 없으면 `0 / 0`으로 표시된다.
4. 현재 PPT 미리보기
    1. 찬양/성경은 텍스트 슬라이드로 표시된다.
    2. 찬송가는 악보 이미지 슬라이드로 표시된다.
    3. 설교, 광고, 커스텀은 각 모듈 전용 레이아웃으로 표시된다.
5. 미리보기 썸네일
    1. 모듈 그룹별로 묶여 표시된다.
    2. 각 그룹에는 모듈 이름과 해당 모듈의 슬라이드 수가 표시된다.
    3. 썸네일을 클릭하면 해당 슬라이드와 모듈이 선택된다.

### 1.5 보조 화면 및 공통 UI

1. 로그인 화면
    1. 이메일 입력 필드와 비밀번호 입력 필드를 표시한다.
    2. 로그인 버튼과 돌아가기 버튼을 제공한다.
    3. 현재 구현에서는 입력값을 브라우저 로컬 인증 상태로 저장하는 MVP 흐름이다.
2. 템플릿 저장 모달
    1. 템플릿 이름 입력 필드를 표시한다.
    2. 저장 버튼, 취소 버튼, 닫기 버튼을 제공한다.
3. 공유 모달
    1. 공유 대상 템플릿 이름을 표시한다.
    2. 모듈 수와 슬라이드 수를 표시한다.
    3. 공유 링크 입력 필드를 표시한다.
    4. 링크 복사 버튼과 닫기 버튼을 제공한다.
4. 토스트
    1. 저장, 삭제, 로그인, PPT 생성, 공유 링크 복사 등 주요 결과 안내 문구를 표시한다.
    2. 화면 하단 안내 영역으로 동작한다.

## 2. 모듈 설명 보강

### 2.1 공통 구성

1. 현재 구현은 실제 DB/API가 아니라 코드에 포함된 샘플 데이터 기준으로 동작한다.
2. 모듈 카드 공통 구성
    1. 드래그 핸들
    2. 모듈 이름
    3. 모듈 종류
    4. 해당 모듈에서 생성되는 PPT 장 수
    5. 접기/펼치기 버튼
    6. 모듈 복제 버튼
    7. 모듈 삭제 버튼
3. 슬라이드 분할
    1. 찬양과 성경 모듈에서 제공된다.
    2. 2줄, 4줄, 직접 입력 중 하나를 선택한다.
    3. 직접 입력 선택 시 편집 가능한 텍스트 입력창이 표시된다.
    4. 직접 입력 내용은 빈 줄, 즉 Enter 두 번 기준으로 새 슬라이드가 생성된다.
4. 폰트
    1. 찬양과 성경의 공통 폰트 섹션에서 제공된다.
    2. 폰트 패밀리 드롭다운을 제공한다.
    3. 크기 숫자 입력을 제공한다.
    4. 굵기 범위 슬라이더를 제공한다.
    5. 자간 범위 슬라이더를 제공한다.
    6. 행간 범위 슬라이더를 제공한다.
    7. 검정/흰색 글자 색상 버튼과 직접 색상 선택을 제공한다.
    8. 왼쪽/가운데/오른쪽 정렬 버튼을 제공한다.
5. 배경
    1. 단색 배경을 선택할 수 있다.
    2. 템플릿 배경은 저장된 템플릿 배경이 아니라 코드에 정의된 프리셋 그라데이션 배경이다.
    3. 이미지 업로드를 통해 해당 모듈의 배경 이미지를 지정할 수 있다.

### 2.2 찬양

1. 설명
    1. 샘플 찬양 데이터에서 곡을 검색하고 선택하여 텍스트 PPT 슬라이드를 생성한다.
2. 모듈 이름
    1. 선택한 찬양 곡명이 모듈 이름으로 표시된다.
3. 구성
    1. 곡 검색
        1. 검색창에 곡명을 입력하면 샘플 찬양 목록에서 일치하는 결과를 표시한다.
        2. 검색 결과를 클릭하면 해당 곡이 선택되고 검색창에 곡명이 입력된다.
        3. 일치하는 곡이 없으면 `검색 결과가 없습니다.`를 표시한다.
        4. 검색 버튼을 클릭하면 현재 검색어 기준으로 결과 목록을 연다.
    2. 슬라이드 분할
        1. 2줄/4줄/직접 입력 방식으로 가사를 슬라이드 단위로 나눈다.
    3. 폰트
        1. 공통 폰트 설정을 적용한다.
    4. 배경
        1. 공통 배경 설정을 적용한다.

### 2.3 찬송가

1. 설명
    1. 샘플 찬송가 데이터에서 곡을 검색하고 선택하여 악보 이미지 PPT 슬라이드를 생성한다.
2. 모듈 이름
    1. 선택한 찬송가 곡명이 모듈 이름으로 표시된다.
3. 구성
    1. 곡 검색
        1. 검색창에 곡명을 입력하면 샘플 찬송가 목록에서 일치하는 결과를 표시한다.
        2. 검색 결과에는 찬송가 메타 정보와 악보 이미지 연결 여부가 표시된다.
        3. 일치하는 곡이 없으면 `검색 결과가 없습니다.`를 표시한다.
    2. 악보 이미지
        1. 선택한 찬송가의 악보 이미지 상태를 표시한다.
        2. PPT 슬라이드는 텍스트가 아니라 악보 이미지 중심으로 생성된다.
4. 특이사항
    1. 현재 찬송가 모듈에는 공통 슬라이드 분할, 폰트, 배경 설정 UI가 표시되지 않는다.
    2. 찬송가 슬라이드 배경은 악보 이미지용 기본 배경으로 생성된다.

### 2.4 성경

1. 설명
    1. 샘플 성경 데이터에서 구약/신약, 번역본, 성경, 장, 절 범위를 선택하여 텍스트 PPT 슬라이드를 생성한다.
2. 모듈 이름
    1. `{성경 이름} {장}:{시작 절}-{끝 절}` 형식으로 표시된다.
3. 구성
    1. 성경 선택
        1. 구약/신약 버튼을 제공한다.
        2. 번역본 드롭다운을 제공한다.
        3. 성경 드롭다운을 제공한다.
        4. 장 드롭다운을 제공한다.
        5. 절 범위 입력 필드를 제공한다.
    2. 절 범위 안내
        1. 현재 선택한 장의 유효 절 범위를 `범위: n-n절` 형식으로 표시한다.
        2. 숫자가 아닌 절 범위를 입력하면 `절 범위를 숫자로 입력하세요. 예: 1-4`를 표시한다.
        3. 시작 절이 끝 절보다 크면 `시작 절은 끝 절보다 작거나 같아야 합니다.`를 표시한다.
        4. 데이터 범위를 벗어나면 `DB에 있는 절 범위는 n-n절입니다.`를 표시한다.
    3. 슬라이드 분할
        1. 2줄/4줄/직접 입력 방식으로 구절을 슬라이드 단위로 나눈다.
    4. 폰트
        1. 공통 폰트 설정을 적용한다.
    5. 배경
        1. 공통 배경 설정을 적용한다.

### 2.5 설교

1. 설명
    1. 설교 제목과 설교 시리즈를 입력하여 설교 제목 슬라이드를 생성한다.
2. 모듈 이름
    1. 설교 제목 입력값이 모듈 이름으로 표시된다.
    2. 설교 제목이 비어 있으면 `설교` 또는 기본 제목으로 표시된다.
3. 구성
    1. 설교 제목
        1. 텍스트 입력 필드로 작성한다.
        2. 입력값은 설교 슬라이드의 주요 제목으로 표시된다.
    2. 제목 폰트
        1. 폰트, 크기, 굵기, 글자 색상, 정렬을 설정한다.
    3. 설교 시리즈
        1. 텍스트 입력 필드로 작성한다.
        2. 입력값은 제목보다 작은 보조 텍스트로 표시된다.
    4. 시리즈 폰트
        1. 폰트, 크기, 굵기, 글자 색상, 정렬을 설정한다.
    5. 배경
        1. 공통 배경 설정을 적용한다.
4. 특이사항
    1. 설교 모듈은 항상 1장의 설교 슬라이드를 생성한다.

### 2.6 광고

1. 설명
    1. 교회 소식 제목 슬라이드와 광고 내용 슬라이드를 생성한다.
2. 모듈 이름
    1. `교회소식` 고정값으로 표시된다.
3. 구성
    1. 광고 항목 목록
        1. 기본 광고 항목 3개가 생성된다.
        2. 각 항목은 접기/펼치기 버튼을 가진다.
        3. 광고 항목이 2개 이상일 때 항목 삭제 버튼이 표시된다.
    2. 광고 내용
        1. 광고 제목 입력 필드를 제공한다.
        2. 광고 제목은 내용 슬라이드의 번호 옆 제목으로 표시된다.
    3. 내용 폰트
        1. 광고 제목의 폰트, 크기, 굵기, 글자 색상, 정렬을 설정한다.
    4. 광고 세부 내용
        1. 광고 세부 내용 입력 영역을 제공한다.
        2. 입력 줄바꿈은 슬라이드에도 줄바꿈으로 반영된다.
    5. 세부 폰트
        1. 광고 세부 내용의 폰트, 크기, 굵기, 글자 색상, 정렬을 설정한다.
    6. 박스 배경색
        1. 광고 항목 박스 배경을 흰색 또는 검은색으로 설정한다.
    7. 광고 추가 버튼
        1. 새 광고 항목을 목록 마지막에 추가한다.
    8. 배경
        1. 공통 배경 설정을 적용한다.
4. 슬라이드 생성
    1. 광고 모듈은 제목 슬라이드 1장을 먼저 생성한다.
    2. 광고 내용은 3개 항목 단위로 내용 슬라이드를 생성한다.
    3. 광고 항목이 4개 이상이면 내용 슬라이드가 추가로 생성된다.

### 2.7 커스텀

1. 설명
    1. 찬양, 찬송가, 성경, 설교, 광고 외에 임의의 제목 또는 내용 슬라이드를 생성한다.
2. 모듈 이름
    1. 제목 슬라이드 타입이면 제목 입력값이 모듈 이름으로 표시된다.
    2. 내용 슬라이드 타입이면 내용 제목 입력값이 모듈 이름으로 표시된다.
3. 구성
    1. 종류
        1. 제목 슬라이드와 내용 슬라이드 중 하나를 선택한다.
    2. 제목 슬라이드
        1. 제목 입력 필드를 제공한다.
        2. 제목 폰트 설정을 제공한다.
        3. 부제목 입력 필드를 제공한다.
        4. 부제목 폰트 설정을 제공한다.
        5. 배경 설정을 제공한다.
    3. 내용 슬라이드
        1. 자주 쓰는 내용 프리셋을 제공한다.
        2. 현재 프리셋은 `사도신경`, `주기도문`, `환영합니다`이다.
        3. 프리셋을 선택하면 제목과 내용이 자동 입력된다.
        4. 제목 입력 필드를 제공한다.
        5. 제목 폰트 설정을 제공한다.
        6. 내용 입력 영역을 제공한다.
        7. 내용 폰트 설정을 제공한다.
        8. 배경 설정을 제공한다.
4. 특이사항
    1. 커스텀 모듈은 선택한 타입에 따라 1장의 슬라이드를 생성한다.
    2. 프리셋 내용을 직접 수정하면 프리셋 선택 상태가 해제된다.

## 3. 템플릿 설명

### 3.1 설명

1. 템플릿은 예배 PPT 제작에 필요한 모듈 구성과 화면 설정을 저장하거나 불러오는 단위이다.
2. 템플릿에는 아래 정보가 포함된다.
    1. 템플릿 이름
    2. 모듈 목록
    3. 모듈 순서
    4. 모듈 종류
    5. 각 모듈의 입력값 및 스타일 설정
    6. 현재 선택된 모듈
    7. 현재 선택된 슬라이드
    8. PPT 비율 설정
    9. 로고 사용 여부, 로고 이미지, 로고 위치
3. 저장된 템플릿은 로그인한 사용자 이메일 기준으로 브라우저 `localStorage`에 저장된다.
4. 로그인하지 않은 상태에서는 저장된 템플릿 목록을 볼 수 없고, 템플릿 저장/불러오기 전 저장이 필요한 경우 로그인 화면으로 이동한다.
5. 추천 템플릿은 코드에 고정된 기본 템플릿이며 사용자가 삭제할 수 없다.

### 3.2 제작탭의 현재 템플릿

1. 템플릿 이름
    1. 제작탭 상단의 입력창에 현재 작업 중인 템플릿 이름이 표시된다.
    2. 템플릿 이름 입력값을 수정하면 현재 작업 상태의 `templateName`이 변경된다.
    3. 입력값이 비어 있으면 기본 이름은 `새로운 템플릿`으로 처리된다.
2. 템플릿 저장 버튼
    1. 상단바의 `템플릿 저장` 버튼과 제작탭 패널의 `템플릿 저장` 버튼은 같은 저장 흐름을 실행한다.
    2. 로그인 상태이면 저장 모달을 열거나 기존 저장 템플릿 덮어쓰기 확인창을 표시한다.
    3. 로그아웃 상태이면 로그인 화면으로 이동하고, 로그인 완료 후 저장 흐름을 이어서 실행한다.

### 3.3 템플릿탭

1. 저장된 템플릿
    1. 로그인한 사용자에게 저장된 템플릿 목록을 표시한다.
    2. 각 템플릿 카드는 아래 요소로 구성된다.
        1. 템플릿 썸네일
        2. 템플릿 이름
        3. 삭제 버튼
    3. 템플릿 썸네일은 해당 템플릿으로 생성되는 첫 번째 슬라이드를 축소 미리보기로 표시한다.
    4. 생성 가능한 슬라이드가 없으면 `미리보기 없음`으로 표시한다.
    5. 저장된 템플릿이 없으면 `저장된 템플릿이 없습니다.`를 표시한다.
    6. 로그아웃 상태이면 `로그인 후 저장한 템플릿을 볼 수 있습니다.`를 표시한다.
2. 추천 템플릿
    1. 코드에 기본 제공되는 추천 템플릿 목록을 표시한다.
    2. 현재 제공되는 추천 템플릿은 아래 2개이다.
        1. `주일예배 기본`
        2. `수요기도회`
    3. 각 추천 템플릿 카드는 아래 요소로 구성된다.
        1. 템플릿 썸네일
        2. 템플릿 이름
    4. 추천 템플릿은 삭제 버튼이 없다.
    5. 추천 템플릿은 미리보기와 적용하기가 가능하다.
3. 템플릿 미리보기
    1. 템플릿 카드를 클릭하면 템플릿 목록 대신 미리보기 화면으로 전환된다.
    2. 미리보기 화면은 아래 요소로 구성된다.
        1. 뒤로가기 버튼
        2. 템플릿 이름
        3. 템플릿 설명
        4. 적용하기 버튼
        5. 해당 템플릿으로 생성되는 슬라이드 미리보기 목록
    3. 추천 템플릿은 코드에 정의된 설명 문구를 표시한다.
    4. 저장 템플릿에 별도 설명이 없으면 모듈 수와 슬라이드 수를 기준으로 설명 문구를 생성한다.

### 3.4 템플릿 저장 데이터

1. 저장 시점의 작업 상태를 템플릿 스냅샷으로 생성한다.
2. 템플릿 스냅샷에는 아래 데이터가 포함된다.
    1. 스키마 버전
    2. 템플릿 이름
    3. 모듈 ID 순서
    4. 모듈 타입 순서
    5. 선택된 모듈 ID
    6. 선택된 슬라이드 ID
    7. PPT 표시 설정
    8. 모듈 전체 데이터
3. 저장된 템플릿 이름이 기존 템플릿 이름과 중복되면 `이름 (1)`, `이름 (2)` 형식으로 자동 변경된다.
4. 기존 저장 템플릿을 덮어쓰는 경우 같은 템플릿 ID를 유지하고 현재 작업 상태로 스냅샷을 갱신한다.

### 3.5 IA와 현재 코드의 차이

1. 현재 코드에는 템플릿 이름을 입력으로 변경하는 처리 함수가 일부 있으나, 템플릿탭 카드 안에서 직접 이름을 변경하는 UI는 렌더링되지 않는다.
2. 현재 템플릿 저장은 서버 저장이 아니라 브라우저 `localStorage` 저장이다.

## 4. 동작

### 4.1 미리보기 동작

1. 미리보기 썸네일 클릭 시
    1. 클릭한 썸네일의 `slideId`가 현재 선택 슬라이드로 저장된다.
    2. 클릭한 썸네일의 `moduleId`가 현재 선택 모듈로 저장된다.
    3. 현재 미리보기 중인 PPT 정보가 선택한 슬라이드 기준으로 변경된다.
    4. 현재 순번 / 전체 PPT 장 수가 변경된다.
    5. 현재 PPT 미리보기 영역이 선택한 슬라이드 내용으로 변경된다.
    6. 제작탭의 선택 모듈 하이라이트도 함께 변경된다.
2. 슬라이드가 없는 경우
    1. 미리보기 영역에 `모듈을 추가하세요`가 표시된다.
    2. 슬라이드 카운터는 `0 / 0` 상태가 된다.
3. 썸네일 영역
    1. 슬라이드는 모듈 그룹별로 묶여 표시된다.
    2. 모듈 순서가 변경되면 썸네일 그룹 순서도 함께 변경된다.

### 4.2 제작탭 - 모듈 동작

1. 모듈 선택 시
    1. 선택한 모듈 ID가 현재 선택 모듈로 저장된다.
    2. 해당 모듈의 첫 번째 슬라이드가 현재 선택 슬라이드로 저장된다.
    3. 선택한 모듈 카드가 하이라이트된다.
    4. 미리보기 영역이 선택한 모듈의 첫 번째 슬라이드로 변경된다.
2. 모듈 추가 버튼 클릭 시
    1. 찬양, 찬송가, 성경, 설교, 광고, 커스텀 버튼 중 클릭한 타입의 새 모듈이 생성된다.
    2. 새 모듈은 모듈 목록 마지막에 추가된다.
    3. 새 모듈이 현재 선택 모듈로 변경된다.
    4. 새 모듈의 첫 번째 슬라이드가 현재 선택 슬라이드로 변경된다.
    5. 전체 모듈 수와 전체 PPT 장 수가 갱신된다.
    6. 미리보기 썸네일 목록이 갱신된다.
    7. `{모듈 종류} 모듈을 추가했습니다.` 안내 문구가 표시된다.
3. 모듈 이동 드래그앤드롭 시
    1. 드래그 중인 모듈 ID가 임시 저장된다.
    2. 다른 모듈 카드 위에 드롭하면 드롭 위치 기준으로 앞 또는 뒤에 삽입된다.
    3. 모듈 순서가 변경된다.
    4. 이동한 모듈이 현재 선택 모듈로 변경된다.
    5. 미리보기 썸네일 그룹 순서가 변경된다.
    6. 모듈 목록 영역의 위/아래 근처에서 드래그하면 자동 스크롤된다.
4. 모듈 복제 버튼 클릭 시
    1. 클릭한 모듈의 정보를 복사한 새 모듈이 생성된다.
    2. 새 모듈은 원본 모듈 바로 아래에 생성된다.
    3. 새 모듈에는 새 ID가 부여된다.
    4. 새 모듈은 펼쳐진 상태로 생성된다.
    5. 검색 결과 열림 상태는 초기화된다.
    6. 복제된 모듈이 현재 선택 모듈로 변경된다.
    7. 복제된 모듈의 첫 번째 슬라이드가 현재 선택 슬라이드로 변경된다.
    8. 전체 모듈 수와 전체 PPT 장 수가 갱신된다.
    9. `{모듈 이름} 모듈을 복제했습니다.` 안내 문구가 표시된다.
5. 모듈 삭제 버튼 클릭 시
    1. `{모듈 이름} 모듈을 삭제할까요?` 확인창이 표시된다.
    2. 확인을 누르면 해당 모듈이 목록에서 제거된다.
    3. 취소를 누르면 아무 동작도 하지 않는다.
    4. 삭제 후 현재 선택 슬라이드는 초기화된다.
    5. 전체 모듈 수와 전체 PPT 장 수가 갱신된다.
    6. 미리보기 썸네일 목록이 갱신된다.
6. 모듈 토글 버튼 클릭 시
    1. 접힌 상태이면 모듈 상세 설정 영역이 펼쳐진다.
    2. 펼쳐진 상태이면 모듈 상세 설정 영역이 숨김 처리된다.
    3. 모듈의 접힘 상태가 저장된다.
7. 모듈 섹션 토글 클릭 시
    1. 슬라이드 분할, 폰트, 배경 등 섹션의 펼침/접힘 상태가 변경된다.
    2. 해당 모듈이 현재 선택 모듈로 변경된다.
    3. 화면이 다시 렌더링된다.

### 4.3 제작탭 - 입력 및 스타일 동작

1. 찬양/찬송가 검색어 입력 시
    1. 검색어가 모듈 상태에 저장된다.
    2. 검색어가 있으면 검색 결과 목록이 열린다.
    3. 검색 결과 영역에 일치하는 곡 목록 또는 결과 없음 문구가 표시된다.
2. 찬양/찬송가 검색 결과 클릭 시
    1. 선택한 곡 ID가 모듈에 저장된다.
    2. 검색창에는 선택한 곡명이 입력된다.
    3. 검색 결과 목록이 닫힌다.
    4. 해당 모듈의 슬라이드 내용과 미리보기가 변경된다.
3. 성경 구약/신약 버튼 클릭 시
    1. 선택한 성경 분류가 저장된다.
    2. 해당 분류의 첫 번째 성경, 첫 번째 장, 기본 절 범위 `1-4`로 초기화된다.
    3. 미리보기와 슬라이드 목록이 변경된다.
4. 성경 번역본/성경/장/절 범위 변경 시
    1. 변경값이 모듈 상태에 저장된다.
    2. 성경 또는 장을 바꾸면 절 범위가 기본값 `1-4`로 초기화된다.
    3. 절 범위가 유효하지 않으면 작업 영역과 미리보기에서 오류 문구가 표시된다.
5. 슬라이드 분할 버튼 클릭 시
    1. 2줄, 4줄, 직접 입력 중 선택한 분할 방식이 저장된다.
    2. 직접 입력 선택 시 원본 내용이 편집 가능한 텍스트로 복사된다.
    3. 슬라이드 목록과 미리보기가 변경된다.
6. 폰트/크기/굵기/색상/정렬 변경 시
    1. 변경한 스타일 값이 모듈 상태에 저장된다.
    2. 해당 모듈이 현재 선택 모듈로 변경된다.
    3. 미리보기와 썸네일이 즉시 갱신된다.
7. 배경색 선택 시
    1. 선택한 단색 또는 템플릿 배경값이 모듈 배경으로 저장된다.
    2. 미리보기와 썸네일이 즉시 갱신된다.
8. 배경 이미지 업로드 시
    1. 업로드한 이미지 파일을 Data URL로 읽는다.
    2. 읽은 이미지가 해당 모듈 배경으로 저장된다.
    3. 미리보기와 썸네일이 갱신된다.
    4. `배경 이미지를 적용했습니다.` 안내 문구가 표시된다.

### 4.4 광고 모듈 동작

1. 광고 항목 토글 클릭 시
    1. 해당 광고 항목의 상세 설정이 펼쳐지거나 접힌다.
    2. 광고 모듈이 현재 선택 모듈로 변경된다.
    3. 광고 내용 슬라이드가 현재 선택 슬라이드로 변경된다.
2. 광고 추가 버튼 클릭 시
    1. 새 광고 항목이 광고 목록 마지막에 추가된다.
    2. 새 항목의 기본 제목은 `광고 {번호}` 형식이다.
    3. 광고 모듈이 현재 선택 모듈로 변경된다.
    4. 광고 내용 슬라이드가 현재 선택 슬라이드로 변경된다.
    5. 광고 항목이 4개 이상이면 3개 항목 단위로 내용 슬라이드가 분할 생성된다.
3. 광고 항목 삭제 버튼 클릭 시
    1. 해당 광고 항목이 삭제된다.
    2. 광고 항목이 모두 삭제되면 기본 광고 항목 1개가 다시 생성된다.
    3. 광고 모듈이 현재 선택 모듈로 유지된다.
    4. 광고 내용 슬라이드와 썸네일이 갱신된다.
4. 광고 제목/세부 내용 입력 시
    1. 입력값이 해당 광고 항목에 저장된다.
    2. 미리보기와 썸네일이 즉시 갱신된다.

### 4.5 커스텀 모듈 동작

1. 제목 슬라이드/내용 슬라이드 선택 시
    1. 선택한 슬라이드 타입이 저장된다.
    2. 커스텀 모듈이 현재 선택 모듈로 변경된다.
    3. 커스텀 모듈의 첫 번째 슬라이드가 현재 선택 슬라이드로 변경된다.
    4. 타입에 맞는 입력 필드와 미리보기가 표시된다.
2. 자주 쓰는 내용 프리셋 클릭 시
    1. 커스텀 모듈 타입이 내용 슬라이드로 변경된다.
    2. 선택한 프리셋 ID, 제목, 내용이 모듈에 저장된다.
    3. 미리보기와 썸네일이 갱신된다.
3. 커스텀 제목/부제목/내용 입력 시
    1. 입력값이 커스텀 모듈에 저장된다.
    2. 프리셋 내용을 직접 수정하면 프리셋 선택 상태는 해제된다.
    3. 미리보기와 썸네일이 즉시 갱신된다.

### 4.6 제작탭 - 초기화 동작

1. 초기화 버튼 클릭 시
    1. 현재 저장 템플릿이 있고 작업 내용이 저장 템플릿과 다르면 `초기화 전에 현재 템플릿에 덮어쓸까요?` 확인창이 표시된다.
    2. 덮어쓰기를 선택하면 현재 저장 템플릿을 현재 작업 상태로 갱신한 뒤 초기화 확인으로 이동한다.
    3. 덮어쓰기를 선택하지 않으면 `변경된 내용을 새 템플릿으로 저장할까요?` 확인창이 표시된다.
    4. 새 템플릿 저장을 선택하면 템플릿 저장 모달을 연다.
    5. 저장 모달에서 저장을 완료하면 초기화 확인창으로 이어진다.
    6. 최종적으로 `현재 작업을 초기 샘플로 되돌릴까요?` 확인창이 표시된다.
    7. 확인을 누르면 기본 상태로 초기화된다.
    8. 취소를 누르면 초기화하지 않는다.
    9. 초기화 완료 시 `초기 샘플로 되돌렸습니다.` 안내 문구가 표시된다.
2. 초기 샘플 상태
    1. 제작탭으로 이동한다.
    2. 템플릿 이름은 `새로운 템플릿`이 된다.
    3. 기본 모듈은 찬양 1개, 성경 1개로 구성된다.
    4. 선택 모듈은 첫 번째 모듈이 된다.
    5. PPT 설정은 기본값으로 돌아간다.

### 4.7 템플릿 저장 동작

1. 템플릿 저장 버튼 클릭 시
    1. 로그인 상태이면 저장 흐름을 실행한다.
    2. 로그아웃 상태이면 로그인 화면으로 이동하고 pending action을 `save`로 저장한다.
    3. 로그인 완료 후 저장 흐름이 이어서 실행된다.
2. 현재 작업이 기존 저장 템플릿에서 시작된 경우
    1. `{템플릿 이름} 템플릿에 덮어쓸까요?` 확인창이 표시된다.
    2. 확인을 누르면 기존 템플릿이 현재 작업 상태로 덮어써진다.
    3. 덮어쓰기 완료 시 `템플릿을 덮어썼습니다.` 안내 문구가 표시된다.
    4. 취소를 누르면 새 템플릿 저장 모달이 열린다.
3. 새 템플릿 저장 모달
    1. 템플릿 이름 입력창에 현재 템플릿 이름이 기본값으로 입력된다.
    2. 입력창에 포커스가 이동하고 텍스트가 선택된다.
    3. 저장 버튼 클릭 시 입력한 이름으로 새 템플릿을 생성한다.
    4. 이름이 비어 있으면 `이름 없는 템플릿`으로 저장한다.
    5. 동일한 이름의 저장 템플릿이 있으면 자동으로 번호를 붙인다.
    6. 새 템플릿은 저장 템플릿 목록의 맨 앞에 추가된다.
    7. 현재 작업의 템플릿 이름과 현재 템플릿 ID가 저장된 템플릿으로 변경된다.
    8. 저장 완료 시 `템플릿을 저장했습니다.` 안내 문구가 표시된다.
4. 저장 모달 닫기/취소 클릭 시
    1. 저장 모달이 닫힌다.
    2. 템플릿 불러오기 또는 초기화 중 저장 흐름이었다면 대기 중인 작업이 취소된다.

### 4.8 템플릿탭 동작

1. 템플릿 메뉴 클릭 시
    1. 작업영역이 템플릿탭으로 전환된다.
    2. 제작탭 패널은 숨김 처리된다.
    3. 템플릿 브라우저가 표시된다.
    4. 메뉴의 활성 상태가 템플릿으로 변경된다.
2. 저장된 템플릿 카드 클릭 시
    1. 클릭한 템플릿의 미리보기 화면으로 이동한다.
    2. 템플릿 이름, 설명, 적용하기 버튼, 슬라이드 미리보기 목록이 표시된다.
3. 추천 템플릿 카드 클릭 시
    1. 클릭한 추천 템플릿의 미리보기 화면으로 이동한다.
    2. 추천 템플릿은 삭제 버튼 없이 미리보기와 적용하기만 제공한다.
4. 뒤로가기 버튼 클릭 시
    1. 템플릿 미리보기 상태가 해제된다.
    2. 저장된 템플릿/추천 템플릿 목록 화면으로 돌아간다.
5. 저장된 템플릿 삭제 버튼 클릭 시
    1. `{템플릿 이름} 템플릿을 삭제할까요?` 확인창이 표시된다.
    2. 확인을 누르면 저장 템플릿 목록에서 삭제된다.
    3. 삭제한 템플릿이 현재 적용 중인 템플릿이면 현재 템플릿 ID가 초기화된다.
    4. 삭제한 템플릿을 미리보기 중이었다면 미리보기 상태가 해제된다.
    5. 삭제 완료 시 `템플릿을 삭제했습니다.` 안내 문구가 표시된다.
    6. 취소를 누르면 아무 동작도 하지 않는다.
6. 적용하기 버튼 클릭 시
    1. 현재 작업에 저장하지 않은 변경사항이 없으면 즉시 템플릿을 불러온다.
    2. 현재 작업에 저장하지 않은 변경사항이 있으면 저장 여부 확인 흐름을 실행한다.
    3. 현재 저장 템플릿이 있고 현재 작업이 저장 템플릿과 다르면 `현재 템플릿에 덮어쓸까요?` 확인창이 표시된다.
    4. 덮어쓰기를 선택하면 현재 템플릿을 갱신한 뒤 선택한 템플릿을 불러온다.
    5. 덮어쓰기를 선택하지 않으면 `변경된 내용을 새 템플릿으로 저장할까요?` 확인창이 표시된다.
    6. 새 템플릿 저장을 선택하면 저장 모달을 열고 저장 완료 후 선택한 템플릿을 불러온다.
    7. 저장하지 않으면 선택한 템플릿을 바로 불러온다.
    8. 불러오기 완료 시 제작탭으로 이동한다.
    9. 템플릿 이름, 모듈 목록, 모듈 순서, 선택 모듈, 선택 슬라이드, PPT 설정이 템플릿 스냅샷 기준으로 변경된다.
    10. 불러오기 완료 시 `템플릿을 불러왔습니다.` 안내 문구가 표시된다.

### 4.9 설정탭 동작

1. 설정 메뉴 클릭 시
    1. 작업영역이 설정탭으로 전환된다.
    2. 제작탭 패널과 템플릿 브라우저는 숨김 처리된다.
    3. 설정 패널이 표시된다.
2. PPT 비율 섹션 토글 클릭 시
    1. PPT 비율 설정 영역이 펼쳐지거나 접힌다.
    2. 접힘 상태가 저장된다.
3. 16:9 버튼 클릭 시
    1. PPT 비율이 `16:9`로 저장된다.
    2. 미리보기 슬라이드 비율이 갱신된다.
    3. PPT 생성 시 16:9 비율이 적용된다.
4. 4:3 버튼 클릭 시
    1. PPT 비율이 `4:3`으로 저장된다.
    2. 미리보기 슬라이드 비율이 갱신된다.
    3. PPT 생성 시 4:3 비율이 적용된다.
5. 로고 섹션 토글 클릭 시
    1. 로고 설정 영역이 펼쳐지거나 접힌다.
    2. 접힘 상태가 저장된다.
6. 로고 ON 버튼 클릭 시
    1. 로고 사용 여부가 켜진다.
    2. 로고 상세 설정 영역이 표시된다.
    3. 로고 이미지가 있으면 미리보기와 PPT 생성 결과에 로고가 표시된다.
7. 로고 OFF 버튼 클릭 시
    1. 로고 사용 여부가 꺼진다.
    2. 로고 상세 설정 영역이 숨김 처리된다.
    3. 미리보기와 PPT 생성 결과에서 로고가 제외된다.
8. 로고 이미지 업로드 시
    1. 이미지 파일만 업로드할 수 있다.
    2. 이미지가 아닌 파일이면 `이미지 파일만 업로드할 수 있습니다.` 안내 문구가 표시된다.
    3. 정상 이미지이면 파일을 Data URL로 읽어 로고 이미지로 저장한다.
    4. 로고 사용 여부가 자동으로 켜진다.
    5. 로고 위치가 없으면 기본 위치로 설정된다.
    6. 적용 완료 시 `로고를 적용했습니다.` 안내 문구가 표시된다.
9. 로고 삭제 버튼 클릭 시
    1. 저장된 로고 이미지가 삭제된다.
    2. `로고를 삭제했습니다.` 안내 문구가 표시된다.
10. 로고 위치 버튼 클릭 시
    1. 좌상단, 우상단, 좌하단, 우하단 중 선택한 위치가 저장된다.
    2. 미리보기와 PPT 생성 결과의 로고 위치가 변경된다.

### 4.10 로그인/로그아웃 동작

1. 로그인 버튼 클릭 시
    1. 로그인 화면으로 이동한다.
    2. 기존 대기 중인 템플릿 불러오기 작업은 초기화된다.
2. 로그인 폼 제출 시
    1. 입력한 이메일을 로그인 사용자 이메일로 저장한다.
    2. 인증 상태가 로그인으로 변경된다.
    3. 해당 이메일 기준의 저장 템플릿 목록을 불러온다.
    4. 에디터 화면으로 돌아간다.
    5. `로그인되었습니다.` 안내 문구가 표시된다.
    6. 로그인 전 대기 중이던 작업이 있으면 이어서 실행한다.
        1. `save`: 템플릿 저장
        2. `save-before-template-load`: 템플릿 불러오기 전 저장 모달 열기
        3. `ppt`: PPT 생성
        4. `share`: 공유 모달 열기
3. 돌아가기 버튼 클릭 시
    1. 로그인하지 않고 에디터 화면으로 돌아간다.
    2. 대기 중인 작업은 취소된다.
4. 로그아웃 버튼 클릭 시
    1. 인증 상태가 로그아웃으로 변경된다.
    2. 저장 템플릿 목록이 비워진다.
    3. 현재 템플릿 ID가 초기화된다.
    4. 템플릿 미리보기 상태와 대기 중인 작업이 초기화된다.
    5. 에디터 화면으로 돌아간다.
    6. `로그아웃되었습니다.` 안내 문구가 표시된다.

### 4.11 PPT 생성 동작

1. PPT 생성 버튼 클릭 시
    1. 로그인 상태이면 PPT 생성 로직을 실행한다.
    2. 로그아웃 상태이면 로그인 화면으로 이동하고 pending action을 `ppt`로 저장한다.
    3. 로그인 완료 후 PPT 생성이 이어서 실행된다.
2. 성경 모듈의 절 범위가 유효하지 않은 경우
    1. 유효하지 않은 성경 모듈이 현재 선택 모듈로 변경된다.
    2. 해당 모듈의 첫 번째 슬라이드가 현재 선택 슬라이드로 변경된다.
    3. 미리보기와 작업영역이 갱신된다.
    4. `성경 범위를 확인하세요: {오류 문구}` 안내 문구가 표시된다.
    5. PPT 파일은 생성되지 않는다.
3. 생성할 슬라이드가 없는 경우
    1. `생성할 슬라이드가 없습니다.` 안내 문구가 표시된다.
    2. PPT 파일은 생성되지 않는다.
4. 정상 생성 시
    1. 현재 모듈 목록 기준으로 슬라이드를 생성한다.
    2. PPT 비율, 배경, 텍스트 스타일, 이미지, 로고 설정을 반영한다.
    3. `worship-ppt-{날짜}.pptx` 파일명으로 다운로드를 실행한다.
    4. 생성 완료 시 `PPT 파일을 생성했습니다.` 안내 문구가 표시된다.

### 4.12 공유하기 동작

1. 공유하기 버튼 클릭 시
    1. 로그인 상태이면 공유 모달을 연다.
    2. 로그아웃 상태이면 로그인 화면으로 이동하고 pending action을 `share`로 저장한다.
    3. 로그인 완료 후 공유 모달이 열린다.
2. 공유 모달 열림 시
    1. 현재 슬라이드 목록을 다시 계산한다.
    2. 현재 템플릿 이름을 표시한다.
    3. `{모듈 수}개 모듈 · {슬라이드 수}장` 형식의 요약 정보를 표시한다.
    4. 공유 링크 입력창에 공유 URL을 표시한다.
    5. 현재 템플릿 ID가 있으면 공유 URL 끝에 해당 ID를 사용한다.
    6. 현재 템플릿 ID가 없으면 공유 URL 끝에 `demo`를 사용한다.
    7. 공유 링크 입력창에 포커스가 이동하고 텍스트가 선택된다.
3. 링크 복사 버튼 클릭 시
    1. 공유 링크 입력창의 URL을 선택한다.
    2. 브라우저 클립보드 API가 있으면 URL을 클립보드에 복사한다.
    3. `공유 링크를 복사했습니다.` 안내 문구가 표시된다.
4. 닫기 버튼 클릭 시
    1. 공유 모달이 닫힌다.

### 4.13 메뉴 동작

1. 제작 메뉴 클릭 시
    1. 제작탭 패널이 표시된다.
    2. 템플릿 브라우저와 설정 패널은 숨김 처리된다.
    3. 메뉴 활성 상태가 제작으로 변경된다.
2. 템플릿 메뉴 클릭 시
    1. 템플릿 브라우저가 표시된다.
    2. 제작탭 패널과 설정 패널은 숨김 처리된다.
    3. 메뉴 활성 상태가 템플릿으로 변경된다.
3. 설정 메뉴 클릭 시
    1. 설정 패널이 표시된다.
    2. 제작탭 패널과 템플릿 브라우저는 숨김 처리된다.
    3. 메뉴 활성 상태가 설정으로 변경된다.
4. My 메뉴 클릭 시
    1. 현재 MVP에서는 비활성 메뉴이다.
    2. 클릭하면 `MVP에서는 제작 메뉴만 사용합니다.` 안내 문구가 표시된다.
    3. 화면 전환은 발생하지 않는다.

## 5. 보강/수정 요약

### 5.1 새로 작성한 섹션

1. `1. 전체화면구성 보강`
    1. 상단바, 메뉴, 작업영역, 미리보기 영역 설명
    2. 로그인 화면, 템플릿 저장 모달, 공유 모달, 토스트 설명
2. `2. 모듈 설명 보강`
    1. 공통 모듈 카드, 슬라이드 분할, 폰트, 배경 설명
    2. 찬양, 찬송가, 성경, 설교, 광고, 커스텀 모듈 설명
3. `3. 템플릿 설명`
    1. 템플릿의 정의
    2. 제작탭 현재 템플릿 설명
    3. 저장된 템플릿/추천 템플릿 설명
    4. 템플릿 미리보기 설명
    5. 템플릿 저장 데이터 설명
4. `4. 동작`
    1. 미리보기 동작
    2. 제작탭 모듈 동작
    3. 입력 및 스타일 동작
    4. 광고/커스텀 모듈 상세 동작
    5. 초기화 동작
    6. 템플릿 저장/삭제/적용 동작
    7. 설정, 로그인/로그아웃, PPT 생성, 공유하기, 메뉴 동작

### 5.2 기존 IA에서 코드와 다른 부분

1. 현재 저장 템플릿은 서버가 아니라 브라우저 `localStorage`에 저장된다.
2. 저장 템플릿은 로그인 이메일별로 분리 저장된다.
3. 템플릿탭에서 저장 템플릿 이름을 직접 수정하는 입력 UI는 현재 렌더링되지 않는다.
4. 현재 찬양, 찬송가, 성경 데이터는 실제 DB/API가 아니라 코드 내 샘플 데이터 기준이다.
5. 찬송가 모듈은 텍스트 편집용 공통 분할/폰트/배경 UI가 아니라 악보 이미지 슬라이드 중심으로 구현되어 있다.

### 5.3 코드에는 있으나 기존 IA에 부족했던 동작

1. 상단바 프로젝트 요약 표시
2. 비활성 My 메뉴와 토스트 안내
3. 로그인 화면, 템플릿 저장 모달, 공유 모달
4. 폰트 자간/행간 설정
5. 설교 제목/시리즈 입력 및 스타일 설정
6. 커스텀 제목/내용 입력 및 프리셋 적용
7. 템플릿 적용 전 저장 여부 확인 흐름
8. 기존 템플릿 덮어쓰기 흐름
9. 초기화 전 저장/덮어쓰기 확인 흐름
10. 로그인 후 pending action 이어서 실행하는 흐름
11. 로고 이미지 업로드/삭제/위치 선택 동작
12. PPT 생성 전 성경 범위 검증
13. 공유 링크 생성 및 복사 동작

### 5.4 이번 범위에서 다루지 않은 항목

1. DB/API 연동 후 실제 찬양, 찬송가, 성경 데이터 조회 방식
2. 서버 기반 템플릿 저장/공유 권한 정책
3. 조직/역할 기반 접근 제어
